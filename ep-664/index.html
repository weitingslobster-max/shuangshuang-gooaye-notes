股癌 EP664：AI 算力變現、Google TPU Cloud、Power Rack 與被動元件瓶頸

依 NotebookLM primary report、Social Worker Daily validated secondary note、supplementary triangulation 與外部查證整理。這是研究筆記，不構成個人化投資建議。

Primary: YouTube / NotebookLMSecondary: Social Worker DailyVerification: external sourcesLanguage: 繁體中文

核心結論

  1. EP664 的主線是「AI 算力變現 → 硬體投資續航 → 電力與被動元件瓶頸」。主持人把 Anthropic 獲利預期、xAI/Colossus 算力出租、Google/Blackstone TPU Cloud 與高功率 AI 機櫃串成同一條邏輯：只要算力可變現，AI capex 的硬體鏈就還沒結束。
  2. Google TPU 從自用走向更外部化，是本集最重要的結構變化。Blackstone 5B 美元、500MW、2027 上線的 TPU Cloud JV 能外部驗證「TPU access 被商品化」；但節目提到的「3:1 自用/外銷」仍需回原始音訊與後續供應鏈資料確認。
  3. 被動元件不是泛泛漲價，而是 AI Power Rack / 高壓 MLCC / Snap-in 鋁電容帶來的規格升級。Murata 與 TrendForce 的外部資料支持高階 MLCC 需求與產品 mix 升級；但 47u/22u 缺貨、信昌電對應、個股漲停仍多屬 source layer。
  4. 台積電與產能限制器的概念是風險管理框架。主持人認為產能不足延緩 AI 泡沫化；若晶圓代工、記憶體、光通、OSAT 產能全部大開，短期營收更好但可能加速供需反轉。
  5. 處置股、健策抄底、信昌電推測等內容必須隔離為操作語境。筆記作者的「抄一點」與 ticker 猜測不是主持人觀點，也不是投資建議。

投資假說草稿

H1. AI 算力變現若持續被驗證，硬體 capex 的尾端瓶頸比 GPU 本身更有彈性

來源依據：Anthropic 獲利預期、xAI 算力出租、Google/Blackstone TPU Cloud。風險：Anthropic profitability 是 investor-shared projection，全年可能因 compute cost 再轉虧；AI capex 若過度融資化，硬體訂單可能突然修正。

H2. 高階 MLCC 與 Snap-in 鋁電容可能進入結構性供需緊張，但個股對應要保守

來源依據：主持人談高壓 MLCC、47u/22u、Power Rack、Snap-in；Murata/TrendForce 支持 AI server 高階 MLCC mix 升級與產能負載。風險：消費電子仍弱，T3 泛用品可能只是恐慌補貨；若大廠擴產，缺貨週期縮短。

H3. TPU 外部化會改變 AI accelerator 估值敘事，但 Google 股價受益可能不如供應鏈明顯

來源依據：Blackstone JV、Broadcom 8-K、主持人談 TPU Cloud 第三方化與 3:1 比例。風險：TPU capacity 是否真正被外部客戶採用、軟體生態、Google 是否願意犧牲 Cloud 控制權仍需觀察。

H4. 處置股與操作日記只能作為風控提醒，不可轉成交易建議

來源依據：主持人中性看待處置股；secondary note 有健策抄一點與信昌電推測。處理方式：流動性被抽離時，盤面可能失真；研究可記錄制度影響，但不能追隨個人操作。
